Core concepts

A local mesh for repository intelligence.

AgentMesh treats each runtime as a native view of the same project-level context graph.

Canonical entities

The v0.1 model has three entity types: root instructions, skills, and subagents. Each runtime maps its native files into that shared shape.

Lockfile state

`agentmesh.lock` records canonical hashes, native emitted hashes, locations, lineage, rename history, and conflict state.

Machine cache

Preserved losing versions, queue records, integrity pins, watcher state, and locks stay in the machine-local cache rather than user-visible runtime files.

Sync pipeline

Scanner, normalizer, merger, emitter.

Every sync pass builds a plan before writing, so conflicts and capability skips are explicit.

Scan

Read canonical files, lockfile state, and detected runtime-native entities.

Normalize

Convert native files into canonical Markdown/TOML-neutral entity payloads.

Merge

Apply structured three-way merge, rename detection, and recoverable conflict records.

Emit

Write canonical files and runtime-native files through adapter-declared boundaries.

See the command reference for operational details.
