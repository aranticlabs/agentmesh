Security model

Explicit trust boundaries for files that agents execute.

AgentMesh is local-first, but hooks run on every relevant runtime event. The security model keeps hook installation explicit and binary identity pinned.

Distribution

Verified release artifacts

Installer channels download platform archives, verify SHA-256 entries from `SHA256SUMS`, and verify the manifest signature and bundle with cosign before installation.

Hooks

Machine-local integrity pins

Hook records point at an absolute binary path and the repository cache records the expected binary hash. `agentmesh upgrade` repins after a deliberate binary change.

Sync

No prompts during sync

Hook, watcher, manual sync, and drainer paths must not stop for prompts. Ambiguous or risky choices are made at init, install, restore, upgrade, or uninstall time.

Recovery

No silent overwrites

Conflicting versions are preserved in the machine-local cache and surfaced through `doctor`, `restore`, and `ack` instead of being discarded.

Risk Mitigation
Unexpected binary on PATH Hook integrity pins store the absolute binary path and expected SHA-256.
Runtime adapter writes outside its surface Adapters declare readable and writable paths, and core enforces boundaries.
Installer tampering Archives are checked against signed checksum manifests before installation.
Destructive uninstall Uninstall removes AgentMesh-owned wiring and keeps purge explicit.
AgentMesh keeps repository files auditable by default.
