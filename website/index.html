Project mesh for AI coding runtimes

One repository brain for every coding agent.

AgentMesh keeps Claude Code and Codex aligned by syncing project instructions, skills, and subagents through a canonical local model and an auditable lockfile.

Install AgentMesh Command reference

Install

Direct installers plus an npm wrapper.

Release installs verify the archive against the checksum manifest and cosign signature before placing the binary.

macOS / Linux

curl -fsSL https://agentmesh.sh/install.sh | sh

Windows

irm https://agentmesh.sh/install.ps1 | iex

npm

npm install -g @aranticlabs/agentmesh

Quickstart

Initialize a repository in under a minute.

Start with inspection, initialize once, then run sync manually or through explicit runtime hooks.

Inspect without writing

agentmesh scan
agentmesh status
agentmesh doctor

Initialize the project mesh

agentmesh init --yes --skip-hooks

Sync and review drift

agentmesh sync
agentmesh diff
agentmesh apply

Install native runtime hooks when ready

agentmesh install --runtime claude
agentmesh install --runtime codex

What ships in v0.1

Bidirectional sync without becoming an inference proxy.

AgentMesh does not run models, store prompts, or proxy API calls. It only keeps repository context coherent across coding runtimes.

Canonical model

Instructions, skills, subagents

Runtime-native files are normalized into a shared `.ai/` entity model and tracked in `agentmesh.lock`.

Non-destructive

Conflicts stay recoverable

Structured merge handles ordinary drift. Losing versions are preserved in the machine-local cache for restore.

Explicit trust

Hooks are installed by command

Runtime hooks are never installed silently. Install, upgrade, uninstall, and restore stay explicit user actions.
